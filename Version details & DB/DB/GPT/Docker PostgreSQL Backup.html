You can back up your PostgreSQL database running in a Docker container to your local machine using the pg_dump command. Follow these steps:

Step 1: Find Your PostgreSQL Container Name

Run the following command to list all running Docker containers:

Look for your PostgreSQL container and note its CONTAINER_ID or NAME.

Step 2: Backup the Database

Use the docker exec command to run pg_dump inside the container and save the backup to a file on your local machine.

sh

  • Replace <container_name> with your PostgreSQL container name.
  • Replace <db_user> with your PostgreSQL username.
  • Replace <db_name> with your database name.
  • The output backup.sql file will be stored in your local machine in the current directory.

Step 3: (Optional) Save the Backup to a Specific Directory

If you want to store the backup in a specific directory, use:

Replace /path/to/backup/ with your preferred directory path.

Alternative: Using docker cp for Backup

You can also create a backup inside the container and copy it to your local machine.

  1. Create the backup inside the container:

    sh

Yes, backup.sql is a valid format for a PostgreSQL backup created using pg_dump. It is a plain-text SQL file containing CREATE TABLE, INSERT, and COPY statements. This format is easy to read and can be restored using psql.

Backup Formats in PostgreSQL

PostgreSQL supports different backup formats using pg_dump:

FormatCommand ExampleDescriptionRestore Command
Plain SQL (.sql)pg_dump -U user -d dbname > backup.sqlA human-readable script containing SQL statements (CREATE, INSERT).psql -U user -d dbname < backup.sql
Custom Format (.dump)pg_dump -U user -d dbname -F c -f backup.dumpCompressed, efficient for large databases, supports parallel restore.pg_restore -U user -d dbname --clean --create backup.dump
Directory (.dir)pg_dump -U user -d dbname -F d -f backup_dir/Stores backup as multiple files in a directory, allows parallel restore.pg_restore -U user -d dbname -d backup_dir/
Tar Format (.tar)pg_dump -U user -d dbname -F t -f backup.tarA tar archive, useful for selective table restoration.pg_restore -U user -d dbname --clean --create backup.tar

Which Format Should You Use?

  • Use backup.sql (Plain SQL) if you want a simple, readable backup that can be restored easily.
  • Use backup.dump (Custom Format) if you need faster restores and advanced options like parallel processing.
